Estratégia de 3 Camadas

1

middleware.ts — Edge Function

Roda em toda requisição antes do Next.js. Verifica cookie authjs.session-token diretamente (sem importar Auth.js para manter bundle < 1MB no Vercel free tier). Redireciona /auth/* rotas para login se cookie ausente.

2

Server Component — auth() check

Cada layout e page protegido chama const session = await auth() do lib/auth.ts. Se session é null, redireciona para /login. Garante proteção mesmo com middleware bypassado.

3

API Route — auth() + userId isolation

Todo Route Handler valida auth() e usa session.user.id em todas as queries Prisma. Garante isolamento de dados — usuário nunca acessa dados de outro tenant.

Providers de Autenticação

Google OAuth 2.0
Configurado no Google Cloud Console. Client ID + Secret em env vars. Redirect URI registrado.
OAuth
Email Magic Link
Resend envia email com token de sessão único. Sem senha. VerificationToken armazenado no DB.
Passwordless

JWT Callbacks (lib/auth.ts)

jwt() callback
Inclui user.id, user.plan e user.trialEndsAt no token JWT para evitar lookup ao DB a cada requisição.
session() callback
Expõe user.id e user.plan via useSession() no cliente e auth() no servidor.
signIn() callback
No primeiro login, seta plan = TRIAL e trialEndsAt = now() + 14 dias.

Feature Gating por Plano

lib/subscription.ts
Exporta PLAN_LIMITS com limites por plano. Funções canCreateMudanca(), canAddItem().
<PaywallGate>
Componente wrapper. Se usuário não tem plano suficiente, renderiza CTA de upgrade em vez do children.
API Plan Check
Route handlers consultam PLAN_LIMITS antes de criar recursos. Retorna 403 com mensagem de upgrade.

Stripe SDK v20 — Notas Importantes

API Version

current_period_end removido
Usar invoice.period_end via latest_invoice em vez do campo da Subscription.
invoice.subscription movido
invoice.parent.subscription_details.subscription
Lazy Proxy em lib/stripe.ts
